# Air-Raid Alert Gateway (Кривий Ріг)

Шлюз повітряних тривог на базі [MeshCore](../../README.md) для Кривого Рогу.

## Що це

Firmware для `companion_radio` (роль Companion, не repeater/room-server), що перетворює стан
повітряної тривоги з інтернету на повідомлення в LoRa-мережі MeshCore — доступне навіть тим,
у кого немає власного інтернету, але є mesh-покриття.

## Що робить

- Опитує [alerts.in.ua](https://alerts.in.ua) (`api.alerts.in.ua/v1/iot/active_air_raid_alerts/<UID>.json`)
  по WiFi кожні 15 секунд.
- При **зміні** стану (тривога ⇄ відбій) шле повідомлення в окремий груповий LoRa-канал
  MeshCore **"Тривога KR"** — його отримують усі учасники mesh-мережі в радіусі покриття,
  незалежно від того, чи є в них інтернет.
- Перше зчитування після старту — беззвучне (встановлює базовий стан, не шле повідомлення).

## Залізо

- **Плата:** LilyGo T3 LoRa32 v1.6.1 (радіочіп SX1276, без TCXO)
- **Діапазон:** 433 МГц
- **Кнопка:** GPIO4 (фізичну кнопку перенесено з GPIO12 — strapping-пін, спричиняв збій монтування
  SPIFFS і хибний вхід у CLI Rescue)

## Радіопараметри

| Параметр | Значення |
|---|---|
| Частота | 433.650 МГц |
| Bandwidth | 62.5 кГц |
| Spreading Factor | 8 |
| Coding Rate | 8 |
| TX Power | 20 dBm |

## Як зібрати

PlatformIO, окремий env:

```
pio run -e LilyGo_TLora_V2_1_1_6_airraid
```

## Configuration

Перед збіркою створіть **свій** `examples/companion_radio/AirRaidGatewayConfig.h` — цей файл
у `.gitignore` і **ніколи не комітиться** (містить секрети):

```cpp
#pragma once

#define GW_WIFI_SSID     "..."   // ваш WiFi (окремо від companion-протоколу — той не зачіпається)
#define GW_WIFI_PASS     "..."

#define ALERTS_TOKEN     "..."   // Bearer-токен, отримати на devs.alerts.in.ua
#define ALERTS_UID       9       // ідентифікатор локації в alerts.in.ua
#define REGION_NAME      "Дніпропетровська обл."

#define CHANNEL_NAME     "Тривога KR"
#define CHANNEL_PSK_HEX  "..."   // 32 hex-символи (128-бітний ключ) — узгодьте з усіма учасниками каналу
```

**Про `ALERTS_UID`:** повний список локацій і їх ID — на
[devs.alerts.in.ua](https://devs.alerts.in.ua). На практиці підтверджений і стабільно робочий
варіант — **UID 9** (Дніпропетровська обл., рівень області): він покриває Кривий Ріг, оскільки
надійного city/hromada-level UID саме для Кривого Рогу не підтверджено (UID 279 перевірявся і
повертає HTTP 404 — не використовуйте його). Якщо знайдете підтверджений city-level UID — тестуйте
обережно: сервіс документує лише коди 401 (поганий/протермінований токен) і 429 (rate limit); 404
означає невірний UID, а не проблему з токеном.

Обмеження запитів alerts.in.ua — 12 req/min; прошивка опитує раз на 15 с, з подвоєнням інтервалу
(до 5 хв) при HTTP 429.

## Функції на пристрої

- OLED-сторінка статусу: велика напис **TRYVOGA** (червоний) / **VIDBIY** (зелений), час останнього
  успішного опитування API, стан WiFi, напруга батареї.
- Екран автоматично прокидається при зміні стану тривоги (і лишається увімкненим ще деякий час),
  навіть якщо до того згас за таймаутом бездіяльності.
- Текст сторінки — латиницею навмисно: стандартний шрифт OLED-дисплея не має кириличних гліфів.
  Повідомлення в самому LoRa-каналі (яке бачать на телефоні/десктопі) лишається кирилицею.

## ⚠️ Застереження

Це **додатковий** канал попередження, а **не заміна** офіційних сирен, застосунку "Повітряна
тривога" чи інших офіційних джерел оповіщення. Дані надходять з неофіційного джерела
(alerts.in.ua), можливі затримки чи збої опитування (WiFi, rate limit, недоступність API).
**Не покладайтеся на цей пристрій як на єдине джерело інформації про повітряну тривогу.**
